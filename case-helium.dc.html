// case study · Helium · 2025

Unifying an ecosystem into one platform

Helium is a decentralized network where users deploy hotspots and earn rewards for contributing coverage. This project unified its fragmented suite of products into a single, cohesive experience designed for everyday users, not just crypto enthusiasts.

ROLE
Lead Product Designer
TIMELINE
2025
FOCUS
Mobile-first
Problem

Onboarding, monitoring, governance, and rewards each lived in separate tools, fragmented and crypto-native by default.

Solution

A unified system with one mental model and a shared design language, built for everyday users.

// discovery

Aligning on outcomes with stakeholders

I worked closely with stakeholders to align on the desired product outcomes and ensure the work met Helium's broader goals.

Key priorities
Expand beyond crypto-native users
Make hotspot onboarding the primary entry point
Clearly surface active hotspots
Enable easy exploration of the network
// design

One system, aligned with the brand

Helium World uses a unified system and visual style aligned with the marketing site, so every action feels consistent, approachable, and clearly part of the same product.

Actionable empty states

Instead of passive empty states, we used them to guide new users toward clear next steps, a direct, approachable path into Helium from the moment they land.

Mobile-first by default

Most users access Helium on mobile, so the platform was designed mobile-first rather than adapted from desktop. Core flows, onboarding a hotspot, splitting rewards with a partner, were built to work clearly on small screens.

A cohesive visual style

A consistent design language set the tone of the brand and unified the product, so every interaction felt native and connected rather than fragmented.
